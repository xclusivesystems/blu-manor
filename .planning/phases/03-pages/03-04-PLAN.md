**Step 1: Create ApplyForm client component**

Create `src/components/apply/ApplyForm.tsx` with `"use client"` directive:

- **State management:** Track form values in `useState` with an object: `{ fullName: '', phone: '', email: '', dob: '', currentSituation: '', supervision: '' }`. Track `errors: Record<string, string>`, `isSubmitting: boolean`, `isSubmitted: boolean`.
- **Fields (all required):**
  1. Full Name — text input
  2. Phone — tel input
  3. Email — email input
  4. Date of Birth — date input
  5. Current Situation — radio group: "Employed" / "Looking for Work"
  6. Supervision Status — radio group: "Yes" / "No"
- **Input styling:** Use design system tokens: `w-full rounded-lg border border-border bg-bg-elevated px-4 py-3 text-sm text-foreground placeholder:text-subtle focus:border-primary focus:outline-none focus:ring-2 focus:ring-primary/20 transition-[border-color,box-shadow] duration-200`. Error state: `border-red-500 focus:border-red-500 focus:ring-red-500/20`.
- **Radio button styling:** Custom radio buttons using design system tokens. Each option as a clickable card/button style (rounded-lg border, highlight when selected with border-primary bg-primary/5).
- **Validation:** On submit, check all fields are non-empty. For email, check basic format (includes @ and .). For phone, check non-empty. Show inline error text below each invalid field in `text-red-400 text-xs mt-1`.
- **Submission:** If valid, set `isSubmitting: true`. Call:
  ```typescript
  const SCRIPT_URL = "https://script.google.com/macros/s/AKfycbyAGohOTYu96rDRWgylkdlCL0clrjcBmlAzEhROw-N_S5PVkpnB-GyPSihKRgIINWSt/exec";

  await fetch(SCRIPT_URL, {
    method: "POST",
    mode: "no-cors",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      timestamp: new Date().toISOString(),
      fullName: formData.fullName,
      dob: formData.dob,
      phone: formData.phone,
      email: formData.email,
      currentSituation: formData.currentSituation,
      supervision: formData.supervision,
    }),
  });
  ```
  After fetch resolves (no-cors = opaque response, treat resolve as success), set `isSubmitted: true`.
- **Success state:** Replace form with a confirmation message: CheckCircle icon (Lucide, text-primary, large), "Application Submitted!", brief message "Thank you for your interest. We'll review your application and contact you within 1-2 business days.", and a Button "Return Home" linking to /.
- **Loading state:** Submit button shows "Submitting..." with `opacity-50 cursor-not-allowed` while `isSubmitting`.
- **Error handling:** Wrap fetch in try/catch. On catch, show a general error message: "Something went wrong. Please call us at 727-563-6540 to complete your application."

**CRITICAL:** Do NOT try to read the response body (res.json() will fail with no-cors). The absence of a thrown exception IS the success signal.
